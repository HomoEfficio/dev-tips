# 가상스레드 특징 및 유의사항

## 잘 언급되지 않는 가상스레드 특징

- FIFO 방식의 ForkJoinPool 사용
- 모든 가상 스레드는 하나의 스레드 그룹에 속한다
- Thread.getAllStackTraces()s는 플랫폼 스레드의 스택 트레이스만 반환하고 가상 스레드의 스택 트레이스는 포함하지 않는다.
- 특정 가상 스레드를 어떤 플랫폼 스레드가 실행하고 있는지 확인할 방법은 없음
  - 가상 스레드는 캐리어 스레드의 ThreadLocal 값을 볼 수 없다
- 스택 프레임을 heap에 저장하여 동적 메모리 관리
- 가상 스레드를 캐리어 스레드에 마운트 할 때 heap에 있는 가상 스레드의 스택 프레임을 캐리어 스레드의 스택으로 임시 복사
- 가상 스레드가 캐리어 스레드로부터 언마운트 될 때는 반대로 캐리어 스레드 스택 프레임 내용이 가상 스레드의 heap에 복사
- ThreadLocal 초기화 부담

## 가상스레드로 전환 시 유의 사항

- 외부 요청 시 rate-limiting
- ThreadLocal 에 따른 메모리 사용량 증가(ScopedValue 사용 고려 필요)
- synchronized 블록에서의 pinning 발생(JDK 24 이상 사용으로 해결 가능)
- native 호출 JFR로 모니터링
